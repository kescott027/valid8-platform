# Risk Register

Format:
- Risk: <description>
- Impact: Low/Med/High
- Likelihood: Low/Med/High
- Mitigation: <what we do>
- Owner: <role>
- Status: Open/Mitigated/Accepted

Initial risks:
- Unsafe repo modifications
- Security regressions due to automation
- Non-deterministic artifact generation
- Model/provider outages
- Cost overruns
