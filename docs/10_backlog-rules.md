# Backlog Rules (summary)

Backlog is generated and maintained under `/project_management/rules/02_backlog-format.md`.

Core constraints:
- Every epic maps to a phase and a measurable outcome
- Every story includes: context, acceptance criteria, implementation notes, tests, security considerations, observability, and rollback plan (when relevant)
- Stories must be sized small enough to finish in <= 2 days of focused work
- Uncertainty triggers a spike story with explicit learning goals and a time box
