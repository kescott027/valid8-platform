# Sprint System

Default cadence:
- 2-week sprints for Phase 1–3
- Phase 0 may run as a fixed checklist sprint

Sprint artifacts:
- Sprint plan in `/project_management/sprints/sprint-XXX.md`
- Stories in `/project_management/stories/`
- Sprint review notes appended to sprint file

Ceremonies:
- Sprint planning (select stories, verify dependencies)
- Mid-sprint health check (scope, blockers)
- Sprint review (demo + evidence)
- Retro (process improvements)

AI responsibilities:
- Keep sprint scope stable unless blocked
- Surface risks early
- Maintain a “blocker board” section in sprint file
